# 戻れメロス トンマナ継承設計書

この設計書は、別ゲームを作るときに「戻れメロス」と同じ雰囲気（トンマナ）と運用を再現するための基準です。

## 1. 非交渉の共通ルール（必ず守る）

- フォント:
  - `PixelMplus12-Regular.ttf`
  - `PixelMplus12-Bold.ttf`
  - 全UIテキスト、ゲーム内テキスト、通知、掲示板、管理画面に適用
- 描画:
  - Canvas は常に `imageSmoothingEnabled = false`
  - CSS の `image-rendering: pixelated` を維持
- 入力:
  - PC: `Space` でアクション
  - スマホ: タップで同アクション
- 文体:
  - ひらがな中心、短文、説明は簡潔
  - 通知文は「〜しました」「〜できません」で統一
- デプロイ:
  - GitHub Pages を本番配信先に固定
  - 本番反映前にローカルプレビューで確認してから push

## 2. 視覚トーン（見た目の仕様）

- 色の方向性:
  - 背景: 指定なし
  - 強調: 指定なし
  - 枠線: 黒/濃灰のピクセル感ある太枠
- UI要素:
  - ボタン: 角張った四角、太い border、疑似的なドロップシャドウ
  - 掲示板: 暗背景 + 明るい文字 + 常時見えるスクロールバー
  - 手紙: 紙色、軽い回転、上部テープ、点線区切り
- タイトル絵:
  - ドット絵
  - 主人公は左右対称で欠けないシルエットを維持

## 3. 音のトーン（Tone.js）

- BGM:
  - 8bit（square系）
  - 軽いループ（テンポ感あり、短周期）
- SE:
  - ミュート切替を常設
- 方針:
  - 派手すぎない音量
  - UI/演出の重要度に応じて短く明快

## 4. 機能トーン（ゲーム外機能）

- タイトル画面:
  - 最新情報が見える（掲示板、通知）
  - スタート前に状況把握できる導線
- 掲示板:
  - 投稿、更新、削除、最新ジャンプ
  - 削除時の確認ダイアログ
  - メンテ時は投稿停止/表示制御
- 手紙:
  - 管理側から更新
  - プレイヤー側で確認チェック
  - 余白バランスを崩さない（見出し・点線周辺の統一）
- 監査:
  - IP/ハッシュ/UAベースの追跡性を維持
  - 管理画面で検索可能

## 5. 推奨アーキテクチャ（新作時）

- `core-style.css`
  - フォント、色定数、ボタン、掲示板、手紙UI
- `core-ui.js`
  - 通知、確認ダイアログ、最新ジャンプ、共通ヘルパー
- `core-chat.js`
  - Nostr購読、投稿、削除、反映
- `core-audio.js`
  - Tone.js 初期化、BGM/SE、ミュート
- `game-xxx.js`
  - ゲーム固有ループとルールのみ

## 6. 実装手順（新作1本の作り方）

1. 既存の見た目共通部（フォント・UI）を先に移植
2. タイトル/掲示板/手紙を表示だけ実装
3. 入力（Space/タップ）と基本ループを接続
4. ゲーム固有ロジックを実装
5. 管理画面連携（手紙・監査）を接続
6. ローカルで見た目/機能確認後に本番 push

## 7. 品質チェックリスト（push前）

- フォントがすべて `PixelMplus12` になっている
- ドット絵がぼけていない
- PC/スマホ両方で操作できる
- 掲示板の投稿・削除・最新ジャンプが動く
- 手紙余白が崩れていない
- `gh auth status` が `kimussarazu` active になっている
- ローカルプレビューで最終確認済み

## 8. 運用ルール（このプロジェクト準拠）

- 本番 push 前に必ずローカルでプレビュー
- OKが出るまで push しない
- push前に `kimussarazu` アカウントを確認
- 変更は小さく分けてコミットする
